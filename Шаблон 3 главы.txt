Глава 3. Разработка прототипа системы биометрической аутентификации на основе голоса
3.1 Постановка задачи

Целью данной главы является разработка прототипа системы высоконадежной биометрической аутентификации, 
использующей голосовые данные пользователя. Основным компонентом данной системы является сверточная нейронная сеть, 
обеспечивающая извлечение биометрических признаков и формирование уникального 256-битного вектора для каждого пользователя.

В рамках данной работы поставлены следующие задачи:

    Определение методов предобработки голосовых данных;

    Проектирование архитектуры сверточной нейронной сети;

    Выбор базы данных для обучения и тестирования модели;

    Обучение модели и получение эмбеддингов пользователей;

    Оценка качества аутентификации с использованием метрик EER, FAR и FRR.

3.2 Предобработка голосовых данных

На вход системы подаются аудиозаписи, содержащие голос пользователя. Для обработки сигналов используется метод выделения мел-частотных кепстральных 
коэффициентов (MFCC), позволяющий получить вектор признаков, отражающих спектральные характеристики речи.

Извлечение MFCC реализуется с использованием библиотеки librosa. Для каждого аудиофрагмента рассчитываются 20 MFCC на отрезке длиной 128 фреймов. 
В случае недостаточной длины сигнала производится дополнение нулями.
3.3 Архитектура нейросетевой модели

Для преобразования MFCC-признаков в уникальный биометрический вектор используется сверточная нейронная сеть. На выходе модели формируется вектор размерности 256,
 соответствующий индивидуальному представлению голосового образца.

Ключевые характеристики модели:

    Вход: матрица признаков размером [20 × 128];

    2 сверточных слоя с активацией ReLU и операцией подвыборки (MaxPooling);

    Полносвязный слой размером 512;

    Выходной слой размерности 256 (эмбеддинг пользователя).

Модель обучается с использованием алгоритма обратного распространения ошибки (backpropagation), оптимизатор — Adam, функция потерь — кросс-энтропия
для классификации.
3.4 База данных голосов

В качестве исходных данных используется открытая база Common Voice (Mozilla) / VoxCeleb / LibriSpeech (выбрать нужное). 
Из базы были выбраны N уникальных пользователей. Для каждого пользователя имеются M записей.

Набор данных делится на:

    Обучающую выборку — 60% записей;

    Валидационную — 20%;

    Тестовую — 20%.

3.5 Обучение модели

Обучение производится на обучающей выборке в течение 20 эпох. В процессе обучения контролируется значение функции потерь, а также 
точность на валидационной выборке.
3.6 Эмбеддинги и процесс аутентификации

После обучения модель применяется для извлечения 256-битных эмбеддингов голосов. Для аутентификации пользователя происходит сравнение 
эмбеддинга нового аудиосигнала с эмбеддингом, сохраненным при регистрации, с помощью косинусного сходства.

Если сходство превышает установленный порог, пользователь считается подлинным.
3.7 Метрики оценки качества

Для оценки эффективности системы используются следующие показатели:

    FAR (False Acceptance Rate) — доля ложных допусков;

    FRR (False Rejection Rate) — доля ложных отказов;

    EER (Equal Error Rate) — значение ошибки при равенстве FAR и FRR;

    Accuracy — доля верных решений (True Accepts + True Rejects).

Также строится ROC-кривая, позволяющая визуализировать соотношение между чувствительностью и специфичностью системы при изменении порога.

В результате экспериментов получены следующие значения:

(Значения после тестирования модели)
3.9 Выводы по главе

В рамках данной главы был реализован прототип системы голосовой аутентификации на основе сверточной нейронной сети. 
Полученные результаты демонстрируют высокий потенциал использования MFCC и CNN для решения задач биометрической идентификации и 
подтверждают работоспособность предлагаемого подхода.
